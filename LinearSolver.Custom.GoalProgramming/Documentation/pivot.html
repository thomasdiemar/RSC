Simplex Pivot — Step-by-Step

A pivot swaps one nonbasic variable into the basis and one basic variable out, by turning the entering column into a unit vector (1 in the pivot row, 0 elsewhere) via Gauss–Jordan row operations. Below are two worked HTML table demos: (1) a degenerate pivot (θ=0) and (2) a non-degenerate pivot (θ>0).

Example A — Degenerate Row Pivot (θ = 0)
Problem (minimization): min z = −(x₁ + x₂)
Equality row (in basis): x₁ − x₂ = 0
Bounds: 0 ≤ x₁ ≤ 0 (fixed at 0), 0 ≤ x₂ ≤ 1
Start: (x₁,x₂)=(0,0)

Initial tableau

BasicRHSx₁x₂Status
x₁ (row)0+1−1B, fixed at 0
z (min)0—r(x₂)=−2
Choose entering (pricing): Minimization ⇒ a nonbasic at LB with r<0 can enter. Here x₂ at LB has r(x₂)=−2 ⇒ enter x₂ from LB.
Augmented Ratio Test (ART): direction in basic space is d = B⁻¹ a₂ = −1. Basics move by Δx_B = −d·θ = +θ in row x₁, i.e., Δx₁=+θ. But x₁ already sits at its UB=0 ⇒ row limit θ_row = 0. Opposite bound of entering: θ_bound = 1 − 0 = 1. Minimum is 0 with a row win ⇒ degenerate row pivot.
Objective update: Δz = r_enter · θ = (−2)·0 = 0 (unchanged).

Gauss–Jordan pivot (θ = 0): normalize pivot row & zero the entering column elsewhere

  1. Normalize row x₁ by −1 → column x₂ becomes 1 in that row.
  2. Zero the x₂ column in all other rows (here: the objective row).
  3. Swap labels: x₂ becomes basic, x₁ leaves (nonbasic at its bound 0).

Tableau after the (degenerate) pivot

x₂ (row)0−1+1B
z (min)0r(x₁) (recomputed)0

Takeaway: The basis changed (x₂ is basic), but the point and z stayed the same (θ=0). New reduced costs may now allow a non-degenerate improving step.

Example B — Non-degenerate Row Pivot (θ > 0)
Problem (minimization): min z = −(x₁ + x₂)
Equality row: x₁ + x₂ = 1
Bounds: 0 ≤ x₁, x₂ ≤ 1
Start: take x₂ basic ⇒ x₂=1, x₁=0

Initial tableau (x₂ basic)

BasicRHSx₁x₂Status
x₂ (row)1+1+1B (at UB=1)
z (min)−1r(x₁)=−10
Choose entering: Minimization ⇒ LB nonbasic with r<0. Here x₁ at LB has r(x₁)=−1 ⇒ enter x₁ from LB.
ART: direction from row is d = B⁻¹ a₁ = +1. Basics change: Δx₂ = −d·θ = −θ. Row limit (x₂ hitting LB=0): θ_row = (1−0)/1 = 1. Opposite bound of entering x₁: θ_bound = 1 − 0 = 1. Tie ⇒ row pivot at θ=1 (non-degenerate).
Objective update: Δz = r_enter · θ = (−1)·1 = −1 ⇒ z: −1 → −2.

Gauss–Jordan pivot (θ = 1)

  1. Normalize row x₂ by +1 → pivot element becomes 1.
  2. Eliminate x₁ from all other rows (objective row).
  3. Swap labels: x₁ becomes basic, x₂ leaves (goes to LB=0).

Tableau after the (non-degenerate) pivot

BasicRHSx₁x₂Status
x₁ (row)1+1+1B
z (min)−20r(x₂)=0

Takeaway: Here the step was positive (θ=1), so the primal point moved from (0,1) to (1,0) and the objective improved by −1. The entering’s reduced cost becomes 0 after the pivot because it is now basic.

Pivot Cheat-Sheet (Gauss–Jordan style)
  1. Pick entering by reduced cost (respect LB/UB sign rules).
  2. ART: compute all row limits and the opposite-bound limit; choose the smallest positive θ.
  3. If opposite bound strictly smallest ⇒ preemptive bound hit (no pivot; update values, recompute reduced costs).
  4. If a row wins/ties ⇒ row pivot:
    • Normalize pivot row so pivot element = 1.
    • Eliminate the entering column in all other rows.
    • Swap basis labels (entering ↔ leaving).
    • Objective change: Δz = r_enter(before) · θ. After pivot, r_enter = 0.
  5. Degenerate pivot: θ=0 but still do the row pivot to change the basis (point and z unchanged).
