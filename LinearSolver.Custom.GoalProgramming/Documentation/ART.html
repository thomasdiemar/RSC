Degenerate Row Pivot in the Bounded Simplex — Full Walkthrough with Simplex Tables

A degenerate row pivot occurs when the Augmented Ratio Test (ART) chooses a step size θ = 0, so the basis changes (we pivot) but the point and objective value do not. Below are minimal, concrete simplex tables that show exactly how and why this happens.

Problem we will use (Minimization)
Objective: min z = −(x₁ + x₂) (equivalently, maximize x₁ + x₂)
Equality row: x₁ − x₂ = 0 (kept inside the simplex, forms the basis row)
Bounds: 0 ≤ x₁ ≤ 0 (i.e., x₁ is fixed at 0), and 0 ≤ x₂ ≤ 1
Start at the feasible point: (x₁, x₂) = (0, 0).

We deliberately make x₁ fixed at its bound so that any attempt to move along the entering column immediately “wants” to push x₁ beyond its allowed value; ART will then return θ=0 → a degenerate row pivot.

Step 0 — Initial Simplex Tableau

BasicRHSx₁x₂Status
x₁ (from row x₁ − x₂ = 0) 0 +1 −1 B (LB=UB=0)
z (min) 0 — r(x₂)=−2
Choose entering variable: We are minimizing, and for a nonbasic at LB we need r < 0. Here x₂ is at LB with r(x₂)=−2 ⇒ eligible. Enter x₂ from LB.
Augmented Ratio Test (ART) for x₂ entering from LB:
  • Direction in basic space: d = B⁻¹ a₂.
  • From the row x₁ − x₂ = 0, the coefficient of x₂ is −1 ⇒ d = −1.
  • Basics change as Δx₁ = −d·θ = −(−1)·θ = +θ, entering changes as Δx₂ = +θ (from LB).
  • Row limit (x₁): x₁ is currently 0 and has UB = 0. Since Δx₁ = +θ would try to increase it, the first bound we’d hit is UB. Formula for LB-entry with d<0 is UB-limit:
    θ_row = (u₁ − x₁)/(-d) = (0 − 0)/(1) = 0.
  • Opposite bound of entering x₂: θ_bound = u₂ − x₂ = 1 − 0 = 1.
The smallest nonnegative limit is θ = 0, and a row wins ⇒ degenerate row pivot.
Objective update: Δz = r_enter × θ = (−2) × 0 = 0. The point and objective stay the same; only the basis will change.

Step 1 — Perform the Degenerate Row Pivot (θ = 0)

We Gauss–Jordan pivot on the (row x₁, column x₂) entry (−1):

  • x₂ becomes basic.
  • x₁ leaves the basis and becomes nonbasic fixed at its bound (0).
  • All numerical values (RHS) remain exactly the same.
BasicRHSx₁x₂Status
x₂ (row) 0 −1 +1 B
z (min) 0 r(x₁) (now nonbasic at 0) —

The basis has changed (which changes the dual quantities / reduced costs), even though the primal point (x₁,x₂)=(0,0) and z=0 did not move. This basis change can unblock progress in subsequent iterations while staying at the same vertex.

Bonus: Degenerate Row Pivot with a Row–Row Tie at θ = 0

We now show a situation where two basic rows produce a zero step limit, so we must choose the leaving row by a tie-break rule (e.g., Bland’s rule).

Variant setup
Objective (min): z = −(x₁ + x₂ + x₃)
Equalities (two rows in the basis):
R₁: x₁ − x₂ = 0 (x₁ basic)
R₂: x₃ − x₂ = 0 (x₃ basic)
Bounds: 0 ≤ x₁ ≤ 0 (fixed at 0), 0 ≤ x₃ ≤ 0 (fixed at 0), 0 ≤ x₂ ≤ 1
Start at (x₁,x₂,x₃)=(0,0,0).

Intuition: increasing x₂ from LB tries to increase both x₁ and x₃ (because of the two coupling rows), but both are fixed at 0 ⇒ both rows block at θ=0.

Choose entering: x₂ at LB with r(x₂)=−3 ⇒ enter x₂ from LB.
ART for x₂:
  • Direction components from the two rows are both d = −1 (coefficients of x₂ in R₁ and R₂).
  • Basics change: Δx₁ = −(−1)θ = +θ, Δx₃ = −(−1)θ = +θ.
  • Row limits:
    • R₁ (x₁ fixed at UB=0): UB-hit immediately ⇒ θ₁ = 0.
    • R₂ (x₃ fixed at UB=0): UB-hit immediately ⇒ θ₂ = 0.
  • Opposite bound of x₂: θ_bound = 1.
Minimum is θ=0 with a row–row tie (R₁ and R₂). Use tie-break (e.g., Bland) to pick the leaving row; say we choose R₁ → leave x₁.
Objective change: Δz = r_enter × θ = (−3) × 0 = 0. (Still degenerate.)

Pivots at θ = 0 (row–row tie)

  • Pivot on (R₁, x₂) first: x₂ becomes basic, x₁ leaves to bound (0); values unchanged.
  • We could later pivot on R₂ similarly if needed (another degenerate pivot) to change which columns are basic.

Again, the point did not move; only the basis changed. You can see how degeneracy can cause multiple θ=0 pivots if several basics are blocking at their bounds. Using Bland’s rule (or lexicographic pivoting) prevents cycling.

What to Remember

  • Degenerate row pivot = ART returns θ=0 and a row wins (or ties).
  • We still do the Gauss–Jordan pivot to change the basis; the primal point and z stay the same.
  • After the pivot, reduced costs change (because the basis changed), which may open a new improving (non-degenerate) step next.
  • With multiple zero row limits, apply a tie-break rule (Bland) to choose the leaving row.
