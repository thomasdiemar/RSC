Preemptive (Lexicographic) Bounded Simplex

Thrusters: r₁=(2,2,0), r₂=(−2,−2,0), r₃=(0,2,2), r₄=(0,−2,−2). All aᵢ=(100,100,100).
Torque map (from r×a): B = [[ 200,-200, 0, 0], [-200, 200,200,-200], [ 0, 0,-200, 200]]
Hence Tx=200(t1−t2), Ty=−200(t1−t2)+200(t3−t4), Tz=−200t3+200t4.

Priority 1 — Torque equalities only; basics: Tx,Ty,Tz

Equalities kept inside the simplex:
R₁: Tx − 200 t1 + 200 t2 = 0   R₂: Ty + 200 t1 − 200 t2 − 200 t3 + 200 t4 = 0   R₃: Tz + 200 t3 − 200 t4 = 0.
Bounds (P1): 0 ≤ t_i ≤ 1. Objective (min): z = −(t1+t2+t3+t4). Start at LB: t=0 ⇒ Tx=Ty=Tz=0 (feasible). Take Tx,Ty,Tz as basics.

Priority-1 wrap-up

Across steps, **every variable** \(Tx,Ty,Tz,t1,t2,t3,t4\) was basic at some stage. The torque structure implies \(t_1=t_2\) and \(t_3=t_4\); driving throttles to UB=1 would give \(\sum t_i=4\) and \(z=-4\) (we proceed to Priority 2 with torque canceled explicitly).

Priority 2 — Torque canceled; add pairwise constraints; basics include Fx,Fy,Fz; thruster bounds tightened

Torque canceled ⇒ constraints: add C₁: t1 − t2 = 0, C₂: t3 − t4 = 0. (These are the algebraic form of “zero torque” for this geometry.) We also keep the original torque rows with Tx=Ty=Tz=0 **locked** (LB=UB=0), which simply ensures those rows remain satisfied as we move on the \(t_1=t_2,\;t_3=t_4\) subspace.
Force rows (as basics): since aᵢ=(100,100,100), R₄: Fx − 100Σt = 0, R₅: Fy − 100Σt = 0, R₆: Fz − 100Σt = 0.
New thruster bounds: 0 ≤ t_i ≤ 0.5. Objective (min): z = −(t1+t2+t3+t4).

How the objective & net-evaluation row is computed (minimization)

  • The objective row lists the current z value (RHS) and **reduced costs** for nonbasics. For minimization, an **LB** nonbasic is improving if r < 0 (a **UB** nonbasic if r > 0).
  • When a variable enters, its reduced cost becomes **0** after the pivot.
  • The change in objective for step size θ is Δz = r_enter (before pivot) × θ. In P2 we had r(t2)=−2 due to the pairwise constraint C₁ making t1 follow t2, so θ=0.5 produced Δz=−1; similarly r(t4)=−2 with C₂, another Δz=−1.

How the Augmented Ratio Test works (bounded feature)

  • For the chosen entering variable, compute all **row limits** (each basic hitting LB/UB) and the entering variable’s **opposite-bound** distance.
  • The **smallest positive** among them wins. If the opposite bound is strictly smallest → **preemptive bound hit** (no pivot). If a row wins or ties → **row pivot** (as in our P2 steps where the pairwise basic hit UB=0.5).
